# Кастомные стратегии

Пайплайн поддерживает не только встроенные стратегии, но и пользовательские объекты.

Для таких стратегий рекомендуется придерживаться контракта [ИнтерфейсСтратегииОтказоустойчивости](ИнтерфейсСтратегииОтказоустойчивости.md).

## Минимальный контракт

Пользовательская стратегия должна:

- иметь публичный метод `Применить(Операция, Параметры = Неопределено, СигналПрерыванияОперации = Неопределено)`
- вернуть результат выполнения операции или собственное значение
- при необходимости пробросить исключение дальше

## Рекомендуемая реализация

Чтобы стратегия одинаково работала и с прямым `Действие`, и с лямбда-выражением, и с вложенным шагом пайплайна, используйте `АдаптерОперацииОтказоустойчивости`.

```bsl
// МояСтратегия.os

#Использовать resilience

Функция Применить(Операция, Параметры = Неопределено, СигналПрерыванияОперации = Неопределено) Экспорт

	АдаптерОперации = Новый АдаптерОперацииОтказоустойчивости();
	ПодготовленнаяОперация = АдаптерОперации.Подготовить(Операция, Параметры);

	Результат = ПодготовленнаяОперация.Запустить(СигналПрерыванияОперации);

	Возврат Результат;

КонецФункции
```

Если в пользовательской стратегии используются обработчики событий, их удобно хранить как [ОбработчикОтказоустойчивости](ОбработчикОтказоустойчивости.md). Этот класс берет на себя нормализацию дополнительных параметров и единый вызов обработчика через `Вызвать(<Контекст>)`.

## Подключение к пайплайну

```bsl
#Использовать resilience

Стратегия = Новый МояСтратегия();

Пайплайн = Новый ПайплайнОтказоустойчивости()
	.ДобавитьСтратегию(Стратегия)
	.ДобавитьПовтор(2, 50);
```

## Когда использовать

Пользовательская стратегия полезна, когда нужно:

- добавить собственный вид обертки вокруг операции
- обогатить результат перед возвратом
- внедрить специализированную логику, которой нет во встроенных стратегиях
