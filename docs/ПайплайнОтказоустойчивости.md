## ДобавитьСтратегию

**Синтаксис:**

```bsl
ДобавитьСтратегию(<Стратегия>)
```

**Параметры:**

| Имя | Тип | Описание |
| -- | -- | -- |
| Стратегия | Произвольный | Объект стратегии, поддерживающий метод `Применить` |

**Возвращаемое значение:**

Тип: ПайплайнОтказоустойчивости.

**Описание:**

Добавляет готовую стратегию в конец пайплайна.

> [!NOTE]
> Порядок важен: первая добавленная стратегия становится внешней оберткой для следующих.
Например, если добавить `Fallback`, затем `Retry`, затем `Timeout`, операция будет выполняться внутри `Timeout`, `Timeout` — внутри `Retry`, а вся цепочка — внутри `Fallback`.

Для пользовательских стратегий рекомендуется придерживаться контракта [ИнтерфейсСтратегииОтказоустойчивости](ИнтерфейсСтратегииОтказоустойчивости.md) и использовать рекомендации из [руководства по кастомным стратегиям](КастомныеСтратегии.md).

## ДобавитьЗапаснойСценарий

**Синтаксис:**

```bsl
ДобавитьЗапаснойСценарий(<Обработчик>, <ДополнительныеПараметры>)
```

**Параметры:**

| Имя | Тип | Значение по умолчанию | Описание |
| -- | -- | -- | -- |
| Обработчик | Действие, Строка, Неопределено | `Неопределено` | Обработчик при ошибке основной операции. Строка трактуется как лямбда-выражение. Обработчик должен возвращать результат запасного сценария |
| ДополнительныеПараметры | Массив, ФиксированныйМассив, Произвольный, Неопределено | `Неопределено` | Дополнительные параметры, которые будут переданы обработчику после контекста ошибки |

**Возвращаемое значение:**

Тип: ПайплайнОтказоустойчивости.

**Описание:**

Добавляет в пайплайн [СтратегияЗапасногоСценария](СтратегияЗапасногоСценария.md). Обработчик должен возвращать значение, которое станет результатом запасного сценария. Если заданы дополнительные параметры, они передаются после контекста ошибки. Подробности о лямбда-выражениях см. в [руководстве](ЛямбдаВыражения.md).

## Запустить

**Синтаксис:**

```bsl
Запустить(<Операция>, <Параметры>, <СигналПрерыванияОперации>)
```

**Параметры:**

| Имя | Тип | Значение по умолчанию | Описание |
| -- | -- | -- | -- |
| Операция | Действие, Строка |  | Пользовательская операция или лямбда-выражение |
| Параметры | Массив, ФиксированныйМассив, Произвольный, Неопределено | `Неопределено` | Параметры, которые будут переданы в операцию |
| СигналПрерыванияОперации | СигналПрерыванияОперации, Неопределено | `Неопределено` | Сигнал кооперативного прерывания, который пробрасывается во внутренние шаги пайплайна |

**Возвращаемое значение:**

Тип: Произвольный.

**Описание:**

Запускает пользовательскую операцию через все зарегистрированные стратегии в порядке их добавления. Вместо `Действие` можно передать лямбда-выражение операции.

Параметры передаются в пользовательскую операцию единообразно.
Если передано `Неопределено`, операция вызывается без параметров.
Одиночное значение передается как первый аргумент, а `Массив` или `ФиксированныйМассив` передаются по позициям.

Если передан `СигналПрерыванияОперации`, стратегии и внутренние обертки пайплайна могут использовать его для кооперативного прерывания в соответствии со своей семантикой. Пользовательская операция не получает этот сигнал отдельным аргументом автоматически.

Подробности о лямбда-выражениях см. в [руководстве](ЛямбдаВыражения.md).

**Пример:**

```bsl
ПараметрыОперации = Новый Массив();
ПараметрыОперации.Добавить("left");
ПараметрыОперации.Добавить("right");

Результат = Пайплайн.Запустить(
    "(ПервыйПараметр, ВторойПараметр) -> Возврат СтрШаблон(""%1:%2"", ПервыйПараметр, ВторойПараметр)",
    ПараметрыОперации
);
```


## ЗапуститьАсинх

**Синтаксис:**

```bsl
ЗапуститьАсинх(<Операция>, <Параметры>, <СигналПрерыванияОперации>, <Длительное>)
```

**Параметры:**

| Имя | Тип | Значение по умолчанию | Описание |
| -- | -- | -- | -- |
| Операция | Действие, Строка |  | Пользовательская операция или лямбда-выражение |
| Параметры | Массив, ФиксированныйМассив, Произвольный, Неопределено | `Неопределено` | Параметры, которые будут переданы в операцию |
| СигналПрерыванияОперации | СигналПрерыванияОперации, Неопределено | `Неопределено` | Сигнал кооперативного прерывания, который пробрасывается во внутренние шаги пайплайна |
| Длительное | Булево | `Ложь` | Длительное задание для которого нужно выделить отдельный поток |

**Возвращаемое значение:**

Тип: [Обещание](https://github.com/sfaqer/async/blob/master/src/internal/Классы/Обещание.os).

**Описание:**

Асинхронно запускает пользовательскую операцию через все зарегистрированные стратегии и возвращает обещание из библиотеки [async](https://github.com/sfaqer/async/).

Вместо `Действие` можно передать лямбда-выражение операции.

Параметры передаются в пользовательскую операцию единообразно.
Если передано `Неопределено`, операция вызывается без параметров.
Одиночное значение передается как первый аргумент, а `Массив` или `ФиксированныйМассив` передаются по позициям.

Сигнал не добавляется в аргументы пользовательской операции автоматически. Он используется пайплайном и стратегиями для кооперативного прерывания в соответствии со своей семантикой.

Подробности о лямбда-выражениях см. в [руководстве](ЛямбдаВыражения.md).

**Пример:**

```bsl
Обещание = Пайплайн.ЗапуститьАсинх(
    "(Имя) -> Возврат СтрШаблон(""Привет, %1!"", Имя)",
    "Мир"
);

Результат = Обещание.Получить();
```
