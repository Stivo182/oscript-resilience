# СигналПрерыванияОперации

## Синтаксис:

```bsl
Новый СигналПрерыванияОперации()
```

## Методы

[ЗапроситьПрерывание](#запроситьпрерывание) </br>
[ПрерываниеЗапрошено](#прерываниезапрошено) </br>

## ЗапроситьПрерывание

**Синтаксис:**

```bsl
ЗапроситьПрерывание(<ПричинаПрерывания>)
```

**Параметры:**

| Имя | Тип | Значение по умолчанию | Описание |
| -- | -- | -- | -- |
| ПричинаПрерывания | Строка, Неопределено | `Неопределено` | Текст причины прерывания |

**Описание:**

Запрашивает кооперативное прерывание операции. Повторные вызовы ничего не меняют.


## ПрерываниеЗапрошено

**Синтаксис:**

```bsl
ПрерываниеЗапрошено()
```

**Возвращаемое значение:**

Тип: Булево.

**Описание:**

Возвращает `Истина`, если прерывание уже было запрошено.


## ПричинаПрерывания

**Синтаксис:**

```bsl
ПричинаПрерывания()
```

**Возвращаемое значение:**

Тип: Строка, Неопределено.

**Описание:**

Возвращает причину прерывания, переданную при вызове `ЗапроситьПрерывание(...)`.


## ПроверитьПрерывание

**Синтаксис:**

```bsl
ПроверитьПрерывание()
```

**Описание:**

Если прерывание не запрошено, метод ничего не делает. Если прерывание уже запрошено, вызывается исключение с причиной прерывания или со стандартным текстом `Выполнение операции прервано.`.

> [!NOTE]
> `СигналПрерыванияОперации` не останавливает уже выполняющийся пользовательский код принудительно. Он нужен для кооперативного взаимодействия: стратегии и внутренние обертки библиотеки могут проверять сигнал и реагировать на него в соответствии со своей семантикой, например не запускать следующий шаг или завершать вызов по сигналу.
>
> Пользовательский код тоже может работать с этим объектом напрямую. Если у операции есть доступ к тому же экземпляру `СигналПрерыванияОперации`, она может периодически вызывать `ПроверитьПрерывание()` или проверять `ПрерываниеЗапрошено()` и завершаться кооперативно.
