# Примеры

В каталоге `samples` собраны небольшие сценарии, которые показывают библиотеку в действии.
Каждый пример можно запускать отдельно и использовать как отправную точку для собственных экспериментов.

## Как запускать

Из корня репозитория:

```bash
oscript samples/ПримерОтказоустойчивости.os
```

По тому же шаблону можно запускать любой другой файл из каталога `samples`.

## Что есть в каталоге

- [ПримерОтказоустойчивости.os](ПримерОтказоустойчивости.os) показывает, как `Повтор`, `Предохранитель` и `ЗапаснойСценарий` работают вместе: первый запуск возвращает резервный результат, а после ожидания следующий вызов завершается успешно.
- [ПримерАсинхронногоЗапуска.os](ПримерАсинхронногоЗапуска.os) показывает `ЗапуститьАсинх()`, получение результата через `Получить()` и обработку ошибки через `Поймать()`.
- [ПримерЛямбдаВыражений.os](ПримерЛямбдаВыражений.os) показывает операцию-лямбду и обработчик-лямбду в одном компактном сценарии.
- [ПримерПайплайнаИзГотовыхСтратегий.os](ПримерПайплайнаИзГотовыхСтратегий.os) показывает сборку пайплайна из заранее настроенных объектов стратегий.
- [ПримерПорядкаСтратегий.os](ПримерПорядкаСтратегий.os) показывает, как порядок `ОграничениеПараллелизма` и `ОграничениеВызовов` меняет итоговое поведение пайплайна.
- [ПримерСтратегииПовтора.os](ПримерСтратегииПовтора.os) показывает повтор по результату, собственный расчет задержки и обработчик повтора.
- [ПримерСтратегииПредохранителя.os](ПримерСтратегииПредохранителя.os) показывает размыкание предохранителя, обработчик размыкания и пробный вызов после ожидания.
- [ПримерСтратегииОграниченияПараллелизма.os](ПримерСтратегииОграниченияПараллелизма.os) показывает асинхронный запуск одного вызова, постановку второго в очередь, отклонение третьего и обработчик переполнения.
- [ПримерСтратегииТаймаута.os](ПримерСтратегииТаймаута.os) показывает обработчик таймаута, запрос `СигналПрерыванияОперации`, который пользовательский код проверяет вручную, и повторный запуск после увеличения лимита ожидания.
- [ПримерСтратегииОграниченияВызовов.os](ПримерСтратегииОграниченияВызовов.os) показывает ограничение количества вызовов в интервале и обработчик превышения лимита.
- [ПримерСтратегииЗапасногоСценария.os](ПримерСтратегииЗапасногоСценария.os) показывает возврат запасного результата вместо проброса ошибки.
- [ПримерКастомнойСтратегии.os](ПримерКастомнойСтратегии.os) показывает подключение собственной стратегии через `ДобавитьСтратегию(...)`.

## Вспомогательные классы

- [Классы/НестабильныйСервис.os](Классы/НестабильныйСервис.os) имитирует сервис, который некоторое количество раз завершается ошибкой, а затем начинает отвечать успешно.
- [Классы/МедленныйСервис.os](Классы/МедленныйСервис.os) имитирует сервис с искусственной задержкой, удобный для примеров со стратегией таймаута.
- [Классы/ПрерываемыйСервис.os](Классы/ПрерываемыйСервис.os) показывает кооперативное прерывание длительной операции через общий `СигналПрерыванияОперации`.
- [Классы/СервисПроверкиГотовности.os](Классы/СервисПроверкиГотовности.os) возвращает промежуточный результат `НЕ ГОТОВО`, чтобы показать повтор по результату, а не только по исключению.
- [Классы/ПрефикснаяСтратегия.os](Классы/ПрефикснаяСтратегия.os) показывает минимальную пользовательскую стратегию с использованием `АдаптерОперацииОтказоустойчивости`.

## С чего начать

Если хочется быстро увидеть взаимодействие нескольких встроенных стратегий, начни с [ПримерОтказоустойчивости.os](ПримерОтказоустойчивости.os).
Если интересуют лямбда-выражения, начни с [ПримерЛямбдаВыражений.os](ПримерЛямбдаВыражений.os).
Если нужен конкретный сценарий, удобнее открыть пример нужной стратегии или отдельный пример асинхронного запуска, готовых стратегий или кастомной стратегии.
